Assets 是什麼

一個彈性的物件資料庫

Assets 用四層結構描述你的世界，再用「參照」把它們連起來。把它想成資料庫、資料夾、卡片與欄位。

  1. Object Schema物件結構一整個資料庫，就是你的 CMDB
  2. Object Type物件類型資料夾／分類：伺服器、筆電、軟體
  3. Object物件一張卡片：Jennifer 的 MacBook
  4. Attribute屬性欄位：序號、狀態、負責人、地點
Reference 參照

物件之間用「參照」相連，資料因此不必重複。辦公室從蒙特婁搬到多倫多，只要改「蒙特婁」這一個物件，所有連到它的筆電自動跟著更新。

Assets 長在 Jira Service Management 裡

Assets 不是另一套要切換的系統。物件透過「Assets 自訂欄位」直接接到工單，事件、請求、變更與資產在同一個畫面連動。

  • 🎫

    工單即時帶資產脈絡

    用 Assets 自訂欄位把物件接到請求／事件／變更。客服一開工單就看到受影響裝置的型號、序號、保固與擁有者，不必另開系統查。

  • ⚙️

    以資產為條件觸發自動化

    用物件的狀態或屬性驅動 Jira 自動化：自動路由、更新狀態、到期通知、指派負責人。

  • 🔎

    跨資產與工單查詢分析

    用 AQL 與報表把工單量、資產健康、成本與合規放在一起看，找出常壞的機型與熱門服務。

舉例

員工送出「筆電螢幕破裂」請求，工單的 Affected laptop 欄位就帶出他的 MacBook Pro（狀態標為 DAMAGED，顯示序號與 2020/09/25 保固到期）。客服不必查就知道是誰、哪一台、保固狀態，直接依 SLA（5 分鐘回應、4 小時解決）處理換機，再由自動化把新舊機狀態同步更新。

運作起來是什麼樣子

情境：一位員工的筆電壞了，需要更換。

  1. 1

    偵測與標記

    JSM 查出受影響筆電，狀態自動改為「damaged」。

  2. 2

    提供替換

    從服務目錄帶出可換機型清單，員工送出請求。

  3. 3

    自動審批

    請求自動路由給該員工的主管核准。

  4. 4

    圖形檢視

    客服用 graph 看該員工所有裝置與位置，依型號＋地點篩出庫存。

  5. 5

    狀態同步

    自動把新機擁有者設為員工、狀態改「in service」，舊機標記「broken」。
